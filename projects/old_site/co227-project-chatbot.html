07-11-2017

Setup botpress.io and deploy  a sample chatbot. The sample bot was not connected to any messaging client. The sample bot perfectly worked.

07-11-2017

We had our first meeting with the project owner. We discussed about the main requirements of the project. The project owner asked us to provide a comparison of APIs we can use on 13th Nov 2017 (Milestone 0).

08-11-2018

Project team gathered and discussed about the overall structure of the project and the things we need to workout before starting the actual implementation.

  • DialogFlow API.
  • Nodejs, mysql connection implementation and querying.
  • Moodle database schema/structure.
  • What are the possible user authentication methods we can use.
  • Botpress.io and email services integration.

The initial design/overall structure of the implementation

09-11-2017

The Natural Language Processing(NLP) APIs Comparison.

NLTK

Link: http://www.nltk.org/

DialogFlow

Link : https://dialogflow.com/

Time consuming to develop from scratch.

Already built-in features (context and intents).

Complexity.

Can focus more on the purpose rather than the design.

 

Therefore, we have decided to use DialogFlow as for our NLP API.

14-11-2017

Initially we have decided to process the user requests in the botpress nodejs server it self. Since it is very time consuming and unreliable to do the NLP parts ourselves, we have decided to forward the all the input requests to dialogflow api and then extract the relevant parameters from the user request and make relevant actions through the botpress nodejs server.

But we found that  we can easily connect the chatting clients directly to the dialogflow api. Therefore, we have changed the initial design of our project and omitted the botpress from the nodejs server since it is unnecessary.

The Basic Structure.

16-11-2017

Implemented DialogFlow completely without using botpress. Easy to implement and we can integrate it with many platforms

Use followUp intents to format the response

Docs  https://dialogflow.com/docs/events

21-11-2017

Upto now we did some background work, and have chosen the most suitable api and methods to use in our bot. From today onwards we will officially start to work on our chat bot InfoBot
